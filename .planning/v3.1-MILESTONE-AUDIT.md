## Phase 16 Parallel-Execution Lesson Learned

Documented but NOT a verdict gate. Worth filing as a GSD tooling enhancement:

**Finding:** When 5 executor agents committed in parallel (Wave 1, all `--no-verify`), a shared `.git/index` staging race caused:
- 16-04 + 16-05 frontend code bundled into commit `5fc45f0` under 16-04's attribution
- 16-05 SUMMARY landed in commit `49ad7c7` alongside 16-01's docs

**Functional impact:** ZERO. All artifacts in HEAD, all tests pass, all gates pass, every plan's deliverables landed correctly.

**Audit-trail impact:** Minor. Commit attribution doesn't strictly match plan ownership in 2 of 12 commits.

**Future recommendation:** Per-executor commit locking (not just `--no-verify`) would prevent. Each parallel executor should serialize at the commit boundary (e.g., file-lock around `git add` + `git commit`) while still allowing parallel work on disjoint files.

**Classification:** Not a v3.1 phase verdict gate. Documented here + in 16-04/16-05 SUMMARYs for tooling improvement consideration.
