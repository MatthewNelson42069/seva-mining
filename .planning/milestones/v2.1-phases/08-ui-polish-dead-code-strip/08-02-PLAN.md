<interfaces>
Tailwind v4 @theme inline binding pattern (from RESEARCH §Pattern 1):
- A `--color-X: var(--X)` line inside `@theme inline { ... }` exposes `bg-X / border-X / text-X / ring-X` as Tailwind utilities.
- The underlying `--X` value is declared under `.dark { ... }` (or `:root` for light) and is resolved at runtime via CSS var indirection.

shadcn naming collision (from UI-SPEC §Color and RESEARCH §Anti-Patterns):
- shadcn already owns `--color-accent` (muted-neutral hover surface). Phase 8 amber tokens MUST use the `brand-` prefix: `--color-brand-accent`, `--color-brand-accent-hover`, `--color-brand-accent-subtle`.

Concrete values to add (from UI-SPEC §Color):
- `--brand-accent: oklch(0.769 0.188 70.08)` (visual ≈ amber-500 / #f59e0b)
- `--brand-accent-hover: oklch(0.828 0.189 84.429)` (visual ≈ amber-400 / #fbbf24)
- `--brand-accent-subtle: oklch(0.769 0.188 70.08 / 0.05)` (visual ≈ amber-500/5)

Current state of target files (verified via grep at planning time):
- DayCell.tsx line 89 already has `hover:border-zinc-700 transition-colors` on the non-today branch.
- SummaryCard.tsx outer wrapper uses `cn('w-full rounded-lg border bg-card shadow-sm', ...)` — needs hover added.
