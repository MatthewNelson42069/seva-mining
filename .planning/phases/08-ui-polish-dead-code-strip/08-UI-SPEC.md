## Typography

UI-03 locks weights and forbids font swap. Geist Variable is the only font.

| Role | Size | Weight | Line height | Tailwind class | Usage |
|------|------|--------|-------------|----------------|-------|
| Display / card heading | 16px | 600 (semibold) | 1.5 (default Tailwind) | `text-base font-semibold` | Card title row (e.g., "Weekly Sweep — May 11 – May 17, 2026") — `SummaryCard:45`, `SweeperCard:62` |
| Section heading (`<h3>`) | 14px | 600 (semibold) | 1.5 | `text-sm font-semibold` | Within-card section labels ("Gold News", "Ontario Law") — `SectionBlock:27`; treat sweep card sections likewise |
| Sub-heading / nav label | 14px | 500 (medium) | 1.5 | `text-sm font-medium` | TabNav labels (TabNav:19 frozen); button labels |
| Body | 14px | 400 (regular) | 1.5 | `text-sm` | Default body copy, markdown paragraphs (`prose-sm`), muted helper text |
| Pill / monospace inline | 12px | 400 (regular) | 1.5 | `font-mono text-xs` | X-handle pill text (D-01); engagement numerics `♥123 ⟲45 💬12` when wrapped in `<code>` |

**Weight differentiation rule (UI-03):**
- Card titles and section headings: weight 600
- Tab labels / button labels / `<strong>` inside markdown: weight 500
- All body, list items, muted text: weight 400
- No use of `font-light` (300) or `font-bold` (700) anywhere in Phase 8 — these would break the 3-weight UI-03 contract

**Monospace numerics treatment for UI-03 (per CONTEXT Claude's Discretion):**
- Engagement counts inside `_build_x_posts_md` output (`♥123 ⟲45 💬12, score=N`) are already plain text inside the bullet. Phase 8 wraps the `score=N` value AND each numeric in `<code>` via the react-markdown `components.code` override so they render in `font-mono text-xs` automatically. No regex pre-processing on the backend.
- Alternative simpler path: leave numerics as plain text and rely only on the pill for monospace differentiation. Planner picks based on visual density at 1440x900 (D-11 QA checklist will eyeball this).

**Body line-height: 1.5 default.** Heading line-height: 1.5 default (small headings at 14-16px don't need 1.2 compression). No explicit `leading-*` overrides in Phase 8.

### Semantic accent token additions (D-05)

These NEW tokens land in `frontend/src/index.css` under `.dark { ... }` AND mirrored into `@theme inline` so they're addressable as Tailwind utilities (`bg-accent`, `text-accent-hover`, `bg-accent-subtle`). **NOTE — naming collision:** shadcn already defines `--accent` and `--color-accent` in `index.css` for its own muted-neutral hover backgrounds (`oklch(0.269 0 0)` in `.dark`). Use `--color-brand-accent`, `--color-brand-accent-hover`, `--color-brand-accent-subtle` (final naming subject to planner confirmation in `08-PLAN.md`; the contract here is "do not reuse `--accent`").
