<objective>
Move all engagement gate thresholds from hardcoded values to DB config keys, so they can be tuned from Settings > Scoring without a deploy.

Purpose: EXEC-02 requires agents read scoring settings from DB at start of each run. Currently, `passes_engagement_gate` in both Twitter and Instagram agents uses hardcoded constants (500 likes, 40000 views, 50 likes watchlist, etc.). This plan makes those thresholds DB-driven using the established `_get_config(session, key, default)` pattern.

Output: Modified `passes_engagement_gate` functions accept threshold parameters, `_run_pipeline` reads thresholds from DB and passes them through, seed scripts include new config key defaults.
</objective>

async def _get_config(self, session: AsyncSession, key: str, default: str) -> str:
    """Read a config value from the Config table, with default fallback."""
    result = await session.execute(
        select(Config.value).where(Config.key == key)
    )
    row = result.scalar_one_or_none()
    return row if row is not None else default
```

def passes_engagement_gate(likes, views=None, is_watchlist=False, *, impression_count=None) -> bool:
    effective_views = impression_count if impression_count is not None else views
    safe_views = effective_views if effective_views is not None else 0
    if is_watchlist:
        return likes >= 50 and safe_views >= 5000
    else:
        return likes >= 500 and safe_views >= 40000
```

From scheduler/agents/instagram_agent.py passes_engagement_gate (current hardcoded values):
```python
def passes_engagement_gate(likes: int, created_at: datetime) -> bool:
    if likes < 200:
        return False
    now = datetime.now(timezone.utc)
    if created_at.tzinfo is None:
        created_at = created_at.replace(tzinfo=timezone.utc)
    age_hours = (now - created_at).total_seconds() / 3600.0
    return age_hours < 8.0
```
