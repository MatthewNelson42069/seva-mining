## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, tight inline padding (`gap-1`, `px-1`) |
| sm | 8px | Compact element spacing, list item gaps (`gap-2`, `space-y-2`) |
| md | 16px | Default element spacing, card padding (`p-4`, `gap-4`) |
| lg | 24px | Section padding, group separation (`p-6`, `space-y-6`) |
| xl | 32px | Layout gaps, page padding (`p-8`) |
| 2xl | 48px | Major section breaks (`py-12`) |
| 3xl | 64px | Page-level empty-state vertical centering (`py-16`) |

Exceptions:
- Sidebar width: 220px (fixed, inherited from existing `Sidebar.tsx`)
- Touch targets (buttons): minimum 32px height (`h-8`) per existing shadcn Button sm variant
- Nav items: `px-3 py-2` (12px / 8px) matching existing `Sidebar.tsx` pattern
- Table row cells: `px-4 py-3` matching shadcn table convention

Source: Inferred from existing `Sidebar.tsx`, `ApprovalCard.tsx`, shadcn component conventions.

## Color

All colors are expressed as shadcn CSS variable tokens. The project uses `oklch` color space in `index.css`. Executors reference token names, not raw oklch values.

| Role | Token | Usage |
|------|-------|-------|
| Dominant (60%) | `bg-background` / `bg-white` | Page background, main content area |
| Secondary (30%) | `bg-secondary` / `bg-muted` / `bg-sidebar` | Cards (`bg-card`), sidebar (`oklch(0.985 0 0)`), table rows, tab panels |
| Accent (10%) | `bg-blue-50 text-blue-700` | Active nav item highlight only (inherited from `Sidebar.tsx`) |
| Destructive | `var(--destructive)` / `text-destructive` | Reject button, delete actions, error states only |

Accent reserved for: active navigation link background only (`bg-blue-50 text-blue-700`). All other interactive elements use `bg-primary` (dark fill) or `variant="outline"` / `variant="ghost"` / `variant="secondary"` from the shadcn Button component.

Semantic indicator colors (not accent):
- Quota bar green: `bg-green-500` (< 60% usage)
- Quota bar yellow: `bg-yellow-500` (60–79% usage)
- Quota bar red: `bg-red-500` (≥ 80% usage)
- Priority alert banner: `bg-amber-50 border-amber-200 text-amber-800` (non-null `priority_alert` field)
- No-story flag / empty state icon: `bg-green-50` with `text-green-500` check icon (matches existing `EmptyState.tsx`)

Borders: `border-gray-100` for sidebar divider, `var(--border)` (`oklch(0.922 0 0)`) for all card and table borders.

Muted text: `text-muted-foreground` (`oklch(0.556 0 0)`) for secondary labels, captions, and timestamps.

Source: `index.css` CSS variable definitions, `Sidebar.tsx`, `EmptyState.tsx`, RESEARCH.md quota bar example.

## Page-by-Page Interaction Contracts

### DigestPage (`/digest`)

**Layout:** Single-column content, centered within main area, max-width `max-w-3xl` matching content-focused pages.

**Header row:** Page title "Daily Digest" (Display size, semibold) + date string (Body size, muted) + Prev arrow button + Next arrow button. Buttons: `variant="outline"` `size="icon"`. "Next" button disabled when `currentDate === latestDate`.

**Date format:** `format(parse(digestDate, 'yyyy-MM-dd', new Date()), 'EEEE, MMMM d, yyyy')` using `date-fns` — example: "Wednesday, April 2, 2026".

**Priority alert banner:** Rendered only when `priority_alert` is non-null. Full-width `bg-amber-50 border border-amber-200 rounded-lg p-4` with `text-amber-800 text-sm`. Label: "Priority Alert" in `font-semibold`. Positioned immediately below header, above all other content.

**Top stories section:** Section heading "Top Stories" (Heading size). Ordered list (`ol`) of up to 5 story items. Each item: left-aligned number in `text-muted-foreground font-mono text-sm`, story headline in `text-sm font-medium`, source link in `text-xs text-muted-foreground hover:underline` opening in new tab. Defensive: `const stories = Array.isArray(digest.top_stories) ? digest.top_stories : []`.

**Queue snapshot section:** Section heading "Queue Snapshot" (Heading size). Three stat cards in a `grid grid-cols-3 gap-4`. Each card: `rounded-lg border bg-card p-4`, platform label in `text-xs text-muted-foreground uppercase tracking-wide`, count in `text-2xl font-semibold`. Platforms: Twitter, Instagram, Content.

**Yesterday summary section:** Section heading "Yesterday" (Heading size). Inline stat row: "Approved: N · Rejected: N · Expired: N" in `text-sm text-muted-foreground`. Defensive: check field existence before rendering.

**Empty / 404 state:** Reuse `<EmptyState>` with heading "No digest available" and body "No digest has been generated for this date." Icon: `Calendar` from lucide.

**Loading state:** Skeleton placeholders using `bg-muted animate-pulse rounded` divs matching each section's approximate height.

### ContentPage (`/content`)

**Layout:** Single-column, `max-w-3xl`.

**Page header:** "Content Review" (Display size, semibold) + today's date (Body, muted).

**Branch logic (in render order):**

1. Loading: skeleton.
2. Error (non-404): "Failed to load content. Check your connection and try again." with a Retry button (`variant="outline"`).
3. No bundle (404 caught and returned as `null`): `<EmptyState>` heading "No content today" body "The Content Agent has not run yet or found no qualifying story."
4. Bundle with `no_story_flag=true`: `<EmptyState>` heading "No strong story found today" body `No story cleared the 7.0/10 threshold. Score: {bundle.credibility_score?.toFixed(1) ?? '—'}/10.` Icon: `TrendingDown` from lucide.
5. Bundle found, DraftItem found (status=pending): full review UI — see below.
6. Bundle found, no pending DraftItem (already actioned): read-only bundle info with status badge.

**Full review UI:** Two-column on wide viewports (`grid grid-cols-[1fr_300px] gap-6`), stacks to single column below 1024px (desktop-only project so this is sufficient).

Left column — Content details:
- Format badge: `<Badge variant="outline">` with value from `bundle.format_type` (e.g. "Thread", "Long Form", "Infographic").
- Headline / title: Body text if available from `draft_content`.
- Draft content display:
  - `thread`: numbered tweet list, each tweet in a `border rounded-md p-3 text-sm` block.
  - `long_form`: `<Textarea readOnly>` with full post text, `rows={8}`.
  - `infographic`: `<InfographicPreview draft={...} />` component (see below).
- Sources section: heading "Sources" (Label size, muted, uppercase). List of corroborating sources from `bundle.deep_research` — each as `<a href={url} target="_blank" className="text-sm hover:underline text-blue-600">` with source domain shown.
- Rationale section: heading "Rationale" (Label size, muted, uppercase). `<p className="text-sm text-muted-foreground">`.

Right column — Action panel:
- Credibility score: `<ScoreBadge score={bundle.credibility_score} />`.
- `<Button className="w-full">Approve — Copy to Clipboard</Button>` — calls `approveItem(draftItem.id)` then `navigator.clipboard.writeText(getClipboardText(bundle))` then `toast.success("Approved — copied to clipboard")`.
- `<Button variant="outline" className="w-full mt-2">Reject</Button>` — reveals inline rejection reason textarea + confirm button.

**Read-only state (already actioned):** Show bundle info without action buttons. Status badge: `<Badge variant={status === 'approved' ? 'default' : 'secondary'}>Approved / Rejected</Badge>`.

### SettingsPage (`/settings`)

**Layout:** Full-width within main content area. Page heading "Settings" (Display size, semibold) + `<Separator className="my-4" />`. Then `<Tabs defaultValue="watchlists">` with `<TabsList>` containing 6 triggers.

Tab trigger labels: "Watchlists" | "Keywords" | "Scoring" | "Notifications" | "Agent Runs" | "Schedule"

**Watchlists tab:**
- Platform sub-filter: two `<Button variant="outline" size="sm">` toggle buttons (Twitter / Instagram). Active state: `variant="default"`.
- Table columns: Account Handle, Platform, Relationship Value, Notes, Active, Actions.
- `Active` column: read-only display (`Yes` / `No` in `text-sm`). Toggle via edit flow.
- Actions: "Edit" (opens inline row edit) + "Delete" (opens `<Dialog>` confirm).
- Add new row: `<Button size="sm">Add Account</Button>` at top-right of table. Opens an inline form row at the top of the table with `<input>` fields for handle, relationship_value (1–5), notes.
- Delete confirmation dialog: title "Remove Account", body "Remove @{handle} from the {platform} watchlist? This will stop the agent from tracking this account.", buttons: "Cancel" + "Remove" (`variant="destructive"`).

**Keywords tab:**
- Table columns: Term, Platform, Weight, Active, Actions.
- `Active` column: `<input type="checkbox">` styled with `accent-blue-600` for inline toggle — fires `PATCH /keywords/{id}` immediately on change.
- Weight column: `<input type="number" min="0" max="1" step="0.1">` inline field — saves on blur.
- Actions: "Delete" only (terms do not need editing — delete and re-add).
- Add form: inline row at top of table: term text input, platform select (`twitter` / `instagram` / `content`), weight number input (default 1.0). "Add" button.
- Delete confirmation dialog: title "Remove Keyword", body "Remove '{term}' from {platform} keywords?", buttons: "Cancel" + "Remove" (`variant="destructive"`).

**Scoring tab:**
- Grouped into sections by prefix: "Content Agent Scoring" (keys starting with `content_`) and "Twitter Agent Scoring" (keys starting with `twitter_`).
- Each config key: label derived from key name (replace `_` with spaces, title-case), `<input type="number" step="0.1">` or `<input type="number" step="1">` depending on key name (weights → 0.1 step, thresholds → 1 step).
- "Save" button per section (`<Button size="sm">`). Disabled when `isPending`. Only PATCHes dirty fields (fields whose value differs from originally fetched config).
- Success: `toast.success("Scoring settings saved")`.
- Error: `toast.error("Failed to save settings. Try again.")`.

**Notifications tab:**
- Same pattern as Scoring tab but filtered to notification-related config keys (keys containing `whatsapp` or `alert` or `notification`).
- "Save" button per group.

**Agent Runs tab:**
- Quota widget at the top: `<QuotaBar>` component.
- Filter dropdown: `<select>` styled as `border rounded-md text-sm px-3 py-1.5` with options: "All agents", "twitter_agent", "instagram_agent", "content_agent", "senior_agent". Drives `agentName` param.
- Table columns: Agent, Started, Status, Found, Queued, Filtered, Errors.
- `Status` column: `<Badge variant={status === 'success' ? 'default' : 'destructive'}>`.
- `Errors` column: if non-null, show `<Button variant="ghost" size="sm">View</Button>` that opens a `<Dialog>` showing the errors JSON formatted as `<pre className="text-xs font-mono whitespace-pre-wrap">`.
- Dates formatted with `date-fns format(parseISO(run.started_at), 'MMM d, HH:mm')`.
- Empty state (no runs): "No agent runs yet. Runs will appear here once agents start executing."

**Schedule tab:**
- Descriptive intro text: `<p className="text-sm text-muted-foreground mb-4">Schedule changes are saved to the database. They take effect on the next worker restart.</p>`.
- One row per schedule config key (e.g. `content_agent_schedule_hour`, `twitter_agent_interval_hours`, `instagram_agent_interval_hours`).
- Label + number input (`type="number"`) + unit label (e.g. "hour", "hours").
- Single "Save Schedule" button for all schedule fields.

**QuotaBar component:**
```
<div className="border rounded-lg p-4 mb-4">
  <div className="flex items-center justify-between mb-2">
    <span className="text-sm font-medium">X API Monthly Quota</span>
    <span className="text-xs text-muted-foreground">{used.toLocaleString()} / {cap.toLocaleString()} reads</span>
  </div>
  <div className="w-full bg-gray-100 rounded-full h-2">
    <div className="{barColor} h-2 rounded-full transition-all" style={{ width: `{pct}%` }} />
  </div>
  <p className="text-xs text-muted-foreground mt-1">
    Resets {reset_date} · Safety margin: {quota_safety_margin.toLocaleString()}
  </p>
</div>
```
Bar color: `bg-green-500` under 60%, `bg-yellow-500` 60–79%, `bg-red-500` 80%+.
