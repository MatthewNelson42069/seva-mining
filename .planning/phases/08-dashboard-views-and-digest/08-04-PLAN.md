**1. Create `KeywordsTab.tsx`:**

CRITICAL: Use `term` field (NOT `keyword`) — matches backend schema `backend/app/schemas/keyword.py`.

Local state:
- `showAddForm`: boolean
- `deleteTarget`: KeywordResponse | null

Data fetching:
```typescript
const { data: keywords = [], isLoading } = useQuery({
  queryKey: ['keywords'],
  queryFn: () => getKeywords(),
})
```

Mutations:
- `createMutation`: `useMutation({ mutationFn: createKeyword, onSuccess: () => { queryClient.invalidateQueries({ queryKey: ['keywords'] }); setShowAddForm(false) } })`
- `updateMutation`: `useMutation({ mutationFn: ({ id, body }: ...) => updateKeyword(id, body), onSuccess: () => queryClient.invalidateQueries({ queryKey: ['keywords'] }) })`
- `deleteMutation`: `useMutation({ mutationFn: deleteKeyword, onSuccess: () => { queryClient.invalidateQueries({ queryKey: ['keywords'] }); setDeleteTarget(null) } })`

Layout per UI-SPEC Keywords tab:
- Table columns: Term, Platform, Weight, Active, Actions
- `Active` column: `<input type="checkbox" checked={kw.active} onChange={() => updateMutation.mutate({ id: kw.id, body: { active: !kw.active } })} />` — fires PATCH immediately on change
- Weight column: `<input type="number" min={0} max={1} step={0.1} defaultValue={kw.weight ?? 1.0} onBlur={(e) => updateMutation.mutate({ id: kw.id, body: { weight: parseFloat(e.target.value) } })} />` — saves on blur
- Actions: "Delete" only (per UI-SPEC: "terms do not need editing — delete and re-add")
- Add form: inline row at top with:
  - Term text input (`placeholder="keyword term"`)
  - Platform select: `<select>` with options "twitter", "instagram", "content"
  - Weight number input (default 1.0)
  - "Add" button
- Delete confirmation dialog:
  - Title: "Remove Keyword"
  - Body: "Remove '{term}' from {platform} keywords?"
  - Buttons: "Keep Keyword" (dismiss) + "Remove" (`variant="destructive"`)
