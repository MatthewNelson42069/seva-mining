     - List of already-used story slugs (from `_get_used_topics()`)

     - Instruction: "Pick a story NOT in the used list. Return JSON: {\"story_title\": \"...\", \"story_slug\": \"giustra-goldcorp-1994\", \"key_claims\": [\"claim1\", \"claim2\"]}"
   - Parse JSON response, return dict with story_title, story_slug, key_claims
   - On parse failure, return None

6. **`_verify_facts()` method**:
   - For each key_claim (up to 3), run SerpAPI Google News search to verify
   - Use `self.serpapi_client.search({"engine": "google", "q": claim})` in executor
   - Return list of verified claims with sources
   - If no verification found for a claim, note it as "unverified" (do not discard the story)

7. **`_draft_gold_history()` method** — Claude Sonnet call:
   - System prompt: senior gold analyst + dramatic historian voice. "Drama-first storytelling. Lead with the most dramatic moment. The audacity, the risk, the payoff."
   - User prompt includes: story_title, verified facts, story arc instructions
   - Instruction: Produce JSON with the locked gold_history structure:
     ```json
     {
       "format": "gold_history",
       "story_title": "...",
       "story_slug": "...",
       "tweets": ["hook tweet", "tweet 2", "... up to 7"],
       "instagram_carousel": [
         {"slide": 1, "headline": "...", "body": "...(max 15 words)...", "visual_note": "..."},
         ...
       ],
       "instagram_caption": "full caption"
     }
     ```
   - Instagram design system in prompt: background #F0ECE4, text #0C1B32, gold accent #D4AF37, one key moment per slide, max 15 words per slide body
   - Parse JSON, return draft_content dict or None on failure

8. **`_run_pipeline()` method**:
   - Read used topics
   - Call `_pick_story()` — if None, create no-story ContentBundle, return
   - Call `_verify_facts()` on key_claims
   - Call `_draft_gold_history()` with story + verified facts
   - Run compliance check using `check_compliance()` from content_agent (import)
   - Persist ContentBundle with `content_type="gold_history"`, `story_headline=story_title`
   - Call `_add_used_topic()` to track the slug
   - If compliance passed: build DraftItem via `build_draft_item()` from content_agent, call `process_new_items()`
   - Deep_research JSONB stores: `{"verified_claims": [...], "story_slug": "..."}`

9. **`run()` method** — entry point for APScheduler:
   - Create AgentRun with `agent_name="gold_history_agent"`
   - Wrap `_run_pipeline()` in try/except/finally (same pattern as ContentAgent.run())
