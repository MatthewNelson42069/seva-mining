  <action>
    This is a human verification checkpoint. No code changes needed.

    What was built (Plans 01-04):
    - Password login page
    - Tabbed queue (Twitter/Instagram/Content) with badge counts
    - Full-width approval cards showing platform badge, source excerpt, account info, score, draft alternatives as tabs, rationale toggle, urgency sort
    - Approve (auto-copies to clipboard + toast), Edit+Approve (inline editing), Reject (category selection) actions
    - 300ms fade-out animation with 5-second undo window on all actions
    - Content tab with summary card + detail modal
    - Related card badges ("Also on [platform]")
    - "Queue is clear" empty state
    - Desktop-only layout (1280px min-width)
    - Light mode, blue accent, Linear/Notion aesthetic
    - 10-15 seeded gold sector draft items in the database

    Prerequisites for testing:
    1. Start the backend: `cd backend && uvicorn app.main:app --reload`
    2. Seed mock data: `cd backend && python -m scripts.seed_mock_data`
    3. Start the frontend: `cd frontend && npm run dev`
    4. Open http://localhost:5173

    Test flow (16 checks):
    1. Login — Enter the dashboard password. Verify you reach the queue page. Refresh the browser — verify you stay logged in (JWT persists).
    2. Platform tabs — Click Twitter, Instagram, Content tabs. Verify each shows different cards. Verify badge counts are visible.
    3. Card content — On a Twitter card, verify you see: platform badge, source account, follower count, score (X.X/10), source text excerpt, draft alternatives as tabs, action buttons.
    4. Draft tabs — Click between Draft A, Draft B, RT Quote tabs on a card. Verify the draft text switches without the card changing height.
    5. Expand source text — Click on a truncated source excerpt. Verify it expands to show full text.
    6. Rationale toggle — Click "Why this post?" on a card. Verify rationale text appears. Click again to collapse.
    7. Approve — Click Approve on a card. Verify: (a) card fades out, (b) toast appears saying "Approved -- copied to clipboard", (c) paste from clipboard confirms draft text was copied, (d) undo appears in toast for 5 seconds.
    8. Undo — Approve another card, then click Undo in the toast within 5 seconds. Verify the card fades back in.
    9. Inline edit — Click on draft text of a card to enter edit mode. Verify textarea appears with the draft text. Edit the text, then click "Edit + Approve". Verify the edited text is what gets copied to clipboard.
    10. Reject — Click Reject on a card. Verify the rejection panel opens with 5 category radio buttons (off-topic, low-quality, bad-timing, tone-wrong, duplicate) and optional notes. Select a category, optionally add notes, click "Confirm Reject". Verify card fades out with toast.
    11. Source link — Click the original post link. Verify it opens in a new tab.
    12. Related badges — Find a card with "Also on Instagram" or "Also on Twitter" badge. Click it. Verify it switches to that platform tab.
    13. Content tab — Switch to Content tab. Verify cards show headline, format, score. Click a card to open the detail modal. Verify full draft is visible. Close modal with X or Escape.
    14. Empty state — Approve/reject all cards in one tab. Verify "Queue is clear" message appears.
    15. Visual design — Confirm: white background, blue accent buttons, clean spacing, Linear/Notion feel. No dark mode toggle visible. No mobile-responsive behavior (window narrower than 1280px should scroll, not reflow).
    16. Logout — Click logout in the sidebar. Verify you return to login page.
  </action>
