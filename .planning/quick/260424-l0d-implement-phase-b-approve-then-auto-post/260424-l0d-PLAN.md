<objective>
Implement Phase B: user-initiated per-item approve-then-post-to-X flow. The user clicks "Post to X" on a draft card's detail modal, sees a full-preview confirmation, clicks confirm, and the backend atomically posts to X via tweepy v2 OAuth 1.0a and persists the result on a new orthogonal `approval_state` column on `draft_items`. Narrow MVP: text-only {breaking_news, thread} content types. Everything else (quotes, gold_history, media) stays out of scope.

Purpose: deliver the first end-to-end automated posting path for Seva Mining. Proves the atomic approve→post pattern (idempotent, feature-flagged, fully tested with mocked tweepy) so Phase B.5 can trivially register remaining text-only types once the shape is battle-tested.

Output: Alembic 0009 migration + dual-model schema parity + backend x_poster service + new FastAPI route + frontend confirm-modal UX + narrowly rewritten CLAUDE.md row + updated env.example + runbook note on X Developer Portal Read+Write prerequisite.
</objective>
