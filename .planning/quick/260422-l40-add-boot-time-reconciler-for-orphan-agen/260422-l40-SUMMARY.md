# Quick Task 260422-l40: Boot-time Reconciler for Orphan agent_runs Summary

**One-liner:** Boot-time sweep using SQLAlchemy bulk UPDATE marks orphan `agent_runs` rows (status='running', older than 30 min) as 'failed' before APScheduler registers any jobs.

## What Was Built

Added `reconcile_stale_runs(threshold_minutes: int = 30) -> int` to `scheduler/worker.py`. This function is awaited exactly once in `main()` between `upsert_agent_config()` and `build_scheduler(engine)`, ensuring it runs at scheduler startup before any sub-agent jobs are registered.

**Problem solved:** Hard-kill events (Railway redeploy SIGKILL, container OOM, host crash) bypass the `try/finally` block in each sub-agent pipeline, leaving `agent_runs` rows stuck at `status='running'` with `ended_at=NULL` forever. In-process exceptions are already handled by the existing `try/except/finally` in each sub-agent — this sweeps only the orphans those blocks could never reach.

## Changes

### scheduler/worker.py

Three edits:

1. **Imports:** Added `update` to `from sqlalchemy import text, select, update`; added `AsyncSessionLocal` to `from database import engine, AsyncSessionLocal`; added `from models.agent_run import AgentRun`.

2. **New function `reconcile_stale_runs()`:** Async function using `AsyncSessionLocal` to run a bulk `update(AgentRun)` WHERE `status='running'` AND `started_at < now - threshold_minutes`. Sets `status='failed'`, `errors=["scheduler restart — run abandoned (process killed before finally block)"]` (JSONB array), `ended_at=now`. Returns rowcount. Logs at INFO only when count > 0.

3. **Call site in `main()`:** `await reconcile_stale_runs()` inserted between `await upsert_agent_config()` and `global _scheduler = await build_scheduler(engine)`.

### scheduler/tests/test_worker.py

Added `reconcile_stale_runs` to the import block and appended three `@pytest.mark.asyncio` tests:

- `test_reconcile_stale_runs_updates_orphans` — rowcount=2 path; verifies UPDATE targets `agent_runs`, `status='failed'`, errors is a list with "scheduler restart" marker, commit called.
- `test_reconcile_stale_runs_skips_fresh_rows` — rowcount=0 path; verifies function returns 0, execute/commit still called.
- `test_reconcile_stale_runs_custom_threshold` — passes `threshold_minutes=60`; inspects WHERE clause bound parameter to assert cutoff is within ±2s of now - 60min.
