# Quick Task 260506-i65 — Fix rotted gold-news RSS pipeline — SUMMARY

**Shipped:** 2026-05-06
**Branch:** `main` (no quick-task branch — single mechanical fix)

## Why

User asked for an audit: *"do a double check to make sure all gold stuff is 100% accurate and it gets all the most relevant gold news."* Live HTTP probe of all 7 RSS_FEEDS URLs in `scheduler/agents/content_agent.py` revealed **6 of 7 are dead.** The gold news pipeline has been silently relying on SerpAPI alone for months while the RSS half rotted invisibly:

| Original feed | Status | Why dead |
|---|---|---|
| `kitco.com/rss/news.xml` | 404 | Kitco removed public RSS entirely; no replacement found |
| `mining.com/feed/` | ✅ 200 (241 KB) | Only working source |
| `juniorminingnetwork.com/feed` | 403 | Anti-scraping blocks browser UA too |
| `gold.org/goldhub/news/feed` | 404 | World Gold Council removed RSS |
| `bnnbloomberg.ca/feed/` | 404 | |
| `feeds.bloomberg.com/commodities/news.rss` | 404 | Bloomberg killed public commodity RSS |
| `goldseek.com/feed/` | 404 | |

Net effect on production: every daily_summary fire was ingesting ~15-30 candidates from SerpAPI alone instead of ~80-150 from the combined SerpAPI + RSS sweep. The relevance filter still surfaced the top 5 of whatever did come in, so the user-visible output was "fine" — but with severely degraded breadth.

## Fix

Single mechanical change to `scheduler/agents/content_agent.py`:

### `RSS_FEEDS` (7 → 4 verified-live, gold-relevant)

```python
RSS_FEEDS = [
    ("https://www.mining.com/feed/", "mining.com"),
    ("https://www.northernminer.com/feed/", "northernminer.com"),       # NEW
    ("https://goldswitzerland.com/feed/", "goldswitzerland.com"),       # NEW
    ("https://www.fxstreet.com/rss/news", "fxstreet.com"),              # NEW
]
```

All 4 verified live with `curl -s --max-time 12 -L` returning HTTP 200 + valid `application/rss+xml` or `text/xml` content type. Sample headlines confirmed gold-relevant on each:
- **northernminer.com** — *"Gold price rebounds on US-Iran deal hopes"*, *"Lundin reports bonanza intercepts at Fruta del Norte"*
- **goldswitzerland.com** — *"OPEC Just Signalled a Historic Gold Tailwind"*, pure gold commentary (Von Greyerz)
- **fxstreet.com** — *"Gold surges as Iran deal hopes crush the US Dollar, yields"*, gold + macro
- **mining.com** — kept, still flagship source at 241 KB

### `CREDIBILITY_TIERS` (added 3 new domains)

```python
"northernminer.com": 0.8,        # Tier 2 — Canadian mining authority
"goldswitzerland.com": 0.6,      # Tier 3 — gold commentary
"fxstreet.com": 0.6,             # Tier 3 — market analysis
```

Legacy entries (kitco/bloomberg/gold.org/etc.) **kept** — even though their RSS is dead, those domains may still appear in SerpAPI results, and removing them from the tier dict would inappropriately penalise legitimate gold-relevant stories from those publications.
