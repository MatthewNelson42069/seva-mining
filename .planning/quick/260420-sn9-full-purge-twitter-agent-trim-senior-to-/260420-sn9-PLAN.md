# Quick 260420-sn9: Full purge Twitter agent + trim Senior to morning-digest-only

<objective>
Full end-to-end purge of the Twitter Agent (scheduler module, backend router/tests, frontend routes/tabs/components) and radical trim of the Senior Agent down to `run_morning_digest` only (keep helpers, delete dedup + queue cap + breaking-news alerts + expiry sweep + expiry alerts + engagement alerts + process_new_item(s)). The morning digest query continues to aggregate from content_bundles-era rows (it already filters `platform='content'`).

This quick task is a structural twin of **260419-lvy** (Instagram purge — full SUMMARY at `.planning/quick/260419-lvy-full-purge-instagram-agent-content-queue/260419-lvy-SUMMARY.md`). Read lvy first — every layer this plan touches was touched there in the same order with the same verification gates.

Sevamining becomes a **single-agent system (Content only)** after this task. Gold History Agent remains a bi-weekly cron but is the same content-style pipeline (not a separate "agent" in the product sense). Senior continues to run one morning WhatsApp digest per day.

**Critical preservations:**
- `scheduler/pyproject.toml` retains `tweepy[async]>=4.14` (content_agent video_clip needs it)
- `scheduler/config.py::Settings` retains `x_api_bearer_token` + `x_api_key` + `x_api_secret` (content_agent + OAuth 1.0a future use)
- `scheduler/agents/content_agent.py` retains `VIDEO_ACCOUNTS`, `_search_video_clips`, `_draft_video_caption`, and the entire `_run_twitter_content_search` method (misleading name — the function body just calls video_clip search; rename is out of scope)
- Backend + frontend **Alembic** history not modified (draft_items.platform column stays `String(20)` — deleted rows are the only DB change)
- `scheduler/agents/gold_history_agent.py` and `scheduler/agents/content_agent.py` stop calling `process_new_items` after this — stories land in DB directly with no Senior-side pipeline

**Cancels:** `260420-s3b` (earlier scoped-smaller attempt; superseded by this full-purge task).

**Purpose:** Twitter agent was out of alignment with the operator's content focus (manual-approval Twitter drafting added approval load without producing platform-native value). X API Basic tier restricted programmatic replies on 2026-04-20 (summon-only), eliminating the agent's future upgrade path. Senior's non-digest responsibilities (dedup, queue cap, breaking news, engagement alerts, expiry sweep) were designed around Twitter volume — with Content-only ingestion at ~5 stories/run, the queue never saturates and the surface is counterproductive.

**Output:** 5 atomic commits — one per layer for rollback safety — plus post-commit prod DB cleanup and CLAUDE.md + STATE.md updates.
</objective>
