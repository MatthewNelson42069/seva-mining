# Phase 260423-d30 Plan 01: Make sub_quotes Independent (dedup_scope="same_type") Summary

**One-liner:** Added `dedup_scope="same_type"` to `sub_quotes::run_draft_cycle` so pull-quote drafts are no longer blocked by cross-agent dedup, closing the last topology delta in the 7-sub-agent spec.

## What Changed

### Code change — quotes.py (minimal, one kwarg + docstring)

`scheduler/agents/content/quotes.py::run_draft_cycle` received one new keyword argument at the `run_text_story_cycle` call site:

```python
dedup_scope="same_type",
```

The docstring was expanded with d30/vxg/mos decision-ID bullets following the same style established in `infographics.py` (of3).

No other lines in `quotes.py` were touched. `_draft`, `REPUTABLE_SOURCES`, `CONTENT_TYPE`, and `AGENT_NAME` are all byte-for-byte unchanged.

### Test change — test_quotes.py (new sibling test)

`test_run_draft_cycle_passes_dedup_scope` was added after the existing `test_run_draft_cycle_passes_filters`. It uses the `side_effect` capture pattern (same as `test_infographics.py::test_run_draft_cycle_passes_new_kwargs`) which produces explicit failure messages vs. the older `await_args.kwargs` spot-check style. Both tests are kept to document the migration transition explicitly per plan instruction.

The new test asserts:
- `agent_name == "sub_quotes"`
- `content_type == "quote"`
- `callable(draft_fn)`
- `max_count == 1` (vxg guard)
- `source_whitelist is REPUTABLE_SOURCES` (mos guard)
- `dedup_scope == "same_type"` (d30 gate — primary assertion)

## Why This Was Needed

Before d30, `sub_quotes` used the default `dedup_scope="cross_agent"`. This meant if `sub_breaking_news` (cron 08:00) drafted a Reuters article, the `_is_already_covered_today` predicate in the shared pipeline would block `sub_quotes` (staggered later cron) from drafting a pull-quote variant of that same article — even though the content type, visual artifact, and editorial purpose are completely different.

The fix mirrors the same independence model already applied to:
- `sub_infographics` (of3, `dedup_scope="same_type"`)
- `sub_gold_media` and `sub_gold_history` (both naturally independent — different story selection entirely)

With d30 applied, `sub_quotes` now deduplicates only against other `quote`-type drafts within the same day — not against `breaking_news`, `thread`, or `long_form` drafts.
