# Quick Task 260419-t78: Replace Gemini Imagen Infographic Rendering — Summary

**One-liner:** Replaced Gemini Imagen infographic path with a long-running Node.js Recharts SSR subprocess (7 chart types, 2 pure-SVG), driven by a new Pydantic BundleCharts schema validated in content_agent with silent thread-downgrade on failure.

---

## What Was Built

### Task 1 — Pydantic v2 BundleCharts Schema (commit `898909b`)

Created `scheduler/models/chart_spec.py` and `backend/app/models/chart_spec.py` (dual-copy pattern — no shared package).

Key design choices:
- `ChartType` is `StrEnum` (ruff UP042 compliant) — JSON serializes to string value directly
- `BundleCharts.charts` has `Field(max_length=2)` — 1-or-2 chart constraint enforced at validation time
- All list fields use `default_factory=list` (not `[]`) to avoid mutable default sharing
- `width/height` default to 1200x675 (16:9 Twitter) so chart_spec can omit them

9 chart types: `bar`, `horizontal_bar`, `line`, `multi_line`, `area`, `stacked_area`, `stat_callouts`, `comparison_table`, `timeline`

7 tests in `scheduler/tests/test_chart_spec.py` covering: all types parse, max 2 charts, invalid type raises, width/height defaults, caption required, stat callout fields, optional col3.

### Task 2 — Node Chart Renderer Package (commit `8d0ea80`)

**`scheduler/chart_renderer/`** — standalone Node.js package:

- `package.json`: recharts pinned to `2.15.4` exactly with explicit warning comment. `@resvg/resvg-js@2.6.2` for SVG→PNG rasterization.
- `render-chart.js`: long-running `readline` stdin loop. Reads one JSON line per chart spec, dispatches to component, calls `ReactDOMServer.renderToStaticMarkup()`, rasterizes with resvg-js, responds with `{"png_b64": "..."}` or `{"error": "..."}`. Uses `process.stdout.write()` (NOT `console.log`) for protocol reliability. Loads Inter TTF fonts at startup (gracefully degrades if not present).
- `download-fonts.sh`: downloads Inter 4.0 TTF files at Docker build time (Regular + Bold). Not committed to repo.
- `README.md`: documents protocol, chart types, font requirement, version pinning warning.

**7 components** in `chart_renderer/components/`:

| Component | Type | Notes |
|-----------|------|-------|
| BarChart.jsx | Recharts | background rect, title text, CartesianGrid, XAxis, YAxis |
| HorizontalBarChart.jsx | Recharts | `layout="vertical"`, swapped axis types |
| LineChart.jsx | Recharts | handles both `line` and `multi_line` via series_labels |
| AreaChart.jsx | Recharts | handles both `area` and `stacked_area` |
| StatCallouts.jsx | Pure SVG | gold accent (#D4AF37) on first stat only |
| ComparisonTable.jsx | Pure SVG | alternating row backgrounds, optional 3rd column |
| Timeline.jsx | Pure SVG | horizontal axis, alternating labels above/below |

**Critical constraint**: 3 chart types (stat_callouts, comparison_table, timeline) are pure SVG — resvg-js does not support `<foreignObject>` which Recharts requires.

**Dockerfile changes**: Changed base image from `python:3.12-slim` to `nikolaik/python-nodejs:python3.12-nodejs22`. Added `COPY`, `WORKDIR`, `npm ci --production`, and `sh download-fonts.sh` steps. All COPY paths adjusted for Railway repo-root build context.

**`.gitignore` additions**: `scheduler/chart_renderer/fonts/*.ttf` and `scheduler/chart_renderer/node_modules/`

### Task 3 — Python ChartRendererClient (commit `beba088`)

**`scheduler/agents/chart_renderer_client.py`**: asyncio subprocess wrapper.

- `ChartRendererClient.start()`: `asyncio.create_subprocess_exec('node', renderer_path, ...)` with PIPE
- `ChartRendererClient.stop()`: terminate + wait, safe if already stopped
- `ChartRendererClient.render_charts(payload, timeout=10.0)`: iterates `payload.charts`, writes JSON line to stdin, reads response line from stdout via `asyncio.wait_for(readline(), timeout)`. Auto-restarts if process is dead. Returns `list[bytes | None]` — `None` for failures (timeouts, errors, bad JSON).
- Module-level singleton via `get_chart_renderer_client()`
- `render_infographic()` helper for direct JSON-string entry point

**`scheduler/worker.py`**: wired `chart_client.start()` in `main()` before scheduler starts, `chart_client.stop()` in `finally` block. Both wrapped in try/except to prevent startup/shutdown failures from blocking other cleanup.

7 tests in `scheduler/tests/test_chart_renderer_client.py` covering: start/stop lifecycle, render success, render error response, render timeout, auto-restart on dead process, empty png_b64.

### Task 4 — Replace Infographic Gemini Path (commit `de31622`)

**`scheduler/agents/image_render_agent.py`**:
- Removed `infographic` key from `ROLES_BY_FORMAT` entirely (comment explains it's handled by chart renderer, not Gemini)

D-18 silent-fail: all exceptions caught and logged, sets `bundle.rendered_images = []` on failure.
- Quote format path: **completely unchanged** (still Gemini Imagen)

**`scheduler/agents/content_agent.py`**:

- Updated `build_draft_item()` infographic case (reads `twitter_caption`, falls back to `charts[0].title`)

7 tests in `scheduler/tests/test_image_render_agent.py` (new file), 4 tests appended to `scheduler/tests/test_content_agent.py`, 3 tests updated in `scheduler/tests/agents/test_image_render.py`.

### Font Download (Docker build time only)

`download-fonts.sh` downloads Inter TTF at Docker build time. The fonts are not committed to the repo (`.gitignore`). In a fresh clone without a Docker build, `scheduler/chart_renderer/fonts/` will not exist. `render-chart.js` handles this gracefully (logs warning to stderr, continues without font config).

For local testing of the Node renderer: `cd scheduler/chart_renderer && sh download-fonts.sh && npm install`

### Dead Code in `_build_prompt()` (image_render_agent.py)

`_build_prompt()` still contains infographic/instagram_slide branches (lines ~335-383 in the original). These are **never reached** — infographic format now early-returns via `_render_infographic_charts()` before `_build_prompt()` is called. The quote path continues to use `_build_prompt()` correctly. These branches were not in the plan's explicit deletion list and were not removed to minimize diff scope. They can be cleaned up in a future lint pass.
