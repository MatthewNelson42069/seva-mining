<objective>
Redesign `sub_infographics` so that its 12:00 PT daily cron produces the **2 best infographics by score** from the day's gold-sector news — even if `sub_breaking_news` already drafted the same stories earlier in the day. Same-type (infographics-only) dedup prevents double-drafting within the agent itself. The shared `run_text_story_cycle` pipeline grows two new orthogonal kwargs (`sort_by`, `dedup_scope`) with backward-compatible defaults — so all 4 other callers (breaking_news, threads, long_form, quotes) remain byte-for-byte unchanged. Infographics also gains structured `notes` telemetry on its `AgentRun` row (parity with gold_media's gmb schema) so no4's PerAgentQueuePage subtitle lights up automatically.

Purpose: Today's behavior starves infographics of the morning's strongest gold stories (breaking_news runs every 2h and blocks them cross-agent by 12:00 PT), and picks "best 2" by recency rather than score when capped. The user asked for score-based top-N and explicit permission to reuse breaking_news stories. This change is surgical: 6 locked decisions from CONTEXT.md, infographics-only, no schema/migration work, no new deps.

Output: Updated shared pipeline with two new kwargs, updated infographics sub-agent opt-ing into both, updated `_is_already_covered_today` with optional content_type scoping, new structured notes telemetry, plus +3-6 new tests. Single atomic commit on `main`.
