# Quick Task 260420-r18: Long-Form Dashboard Field-Name Fix + Remove X as Quote-Post Source — Summary

Two atomic production fixes shipped together: (1) rename `post_text` → `post` in the long-form dashboard preview so every existing DB bundle renders its actual prose instead of the "No draft content available." placeholder, and (2) delete the hardcoded X-account quote-tweet pipeline from `content_agent.py` so quotes are only sourced from reputable articles per editorial policy.

## What Changed

### Task 1 — Frontend long_form field-name fix (commit `fb53254`)

**Problem:** `LongFormPreview.tsx` read `d.post_text`, but the scheduler drafter writes the field as `post` at `content_agent.py:1261`. Every existing long_form bundle in the DB — of which there are many, and operator confirmed they were all showing the placeholder — rendered "No draft content available." despite having valid 1500+ character prose.

**Fix:** Renamed the frontend field read from `d.post_text` to `d.post` in two places:
- `frontend/src/components/content/LongFormPreview.tsx` line 6 (interface field type) and line 11 (local variable initializer)
- `frontend/src/components/approval/__tests__/ContentDetailModal.test.tsx` line 159 (Test 3 fixture)

The local variable name `postText` inside the component was preserved (it's used at lines 31 and 40); only the object key reference changed.

**Why reader-side and not writer-side:** Existing DB rows already have the `post` key. Fixing the frontend retroactively repairs every stale bundle with zero migration. Flipping the backend would only help future rows while leaving historical rows stuck showing the placeholder.

**Verification:**
- `grep -rn "post_text" frontend/src/` → zero matches
- `npm test -- --run ContentDetailModal` → 11/11 tests pass (Test 3 text-content assertion survives the key rename because `getByText(...)` matches rendered prose, not data keys)
- `npm run build` → tsc 0 errors, vite bundled cleanly

### Task 2 — Remove X as a quote-post source (commit `aa7b061`)

**Problem:** Content agent had a hardcoded list of X accounts (`QUOTE_ACCOUNTS` = PeterSchiff, jimrickards, JimRickards, RealJimRogers, RobertKiyosaki) that were scraped via Twitter search every content_agent run to produce `content_type=quote` bundles. This violates the editorial policy that quotes should only come from reputable articles with an institutional citation chain, not from personality-driven X accounts.

**Fix:** Surgical deletion of the entire Twitter-sourced quote pipeline while preserving the article-sourced quote format:

1. **Deleted `QUOTE_ACCOUNTS` module constant** (7 lines, formerly at line 84).
2. **Deleted `_search_quote_tweets()` method** (79 lines, formerly lines 944-1022) — Twitter API search, quota check, like-count filter, result shaping, all removed.
3. **Deleted the `# --- Quote tweets ---` block in `_run_twitter_content_search()`** (59 lines, formerly lines 1863-1921) — the call to `_search_quote_tweets` and its per-tweet for-loop that drafted via `_draft_quote_post`, compliance-checked, persisted as ContentBundle, and enqueued DraftItems. The video-clip block above it is untouched.
4. **Updated comment + docstring** — the `# 11. Twitter video clip + quote search...` comment and the `_run_twitter_content_search` docstring's `"""Search Twitter for video clips and quotes..."""` both had the quote references stripped (now video-clip only).
5. **Enhanced Sonnet format-selection prompt** at what is now line 1152 — appended one sentence to the `"quote"` option line so the LLM more aggressively prefers this format when an article contains a verbatim, attributed statement from a credible named figure (bank analyst, central bank official, WGC, IMF, Fed speaker, etc.) that stands on its own as the story's headline.

**Preserved (critical):**
- `_draft_quote_post()` — still invoked when Sonnet chooses `"quote"` as a format from article content via `_research_and_draft`. Its two existing tests at `test_content_agent.py:939` (`test_quote_draft_returns_three_new_fields`) and `:1613` (`test_draft_quote_post_system_prompt_contains_snapshot_block`) pass unchanged.
- `_search_video_clips()`, `_draft_video_caption()`, `VIDEO_ACCOUNTS` — video-clip Twitter pipeline fully intact.
- `GOLD_KEYWORDS`, `QUOTE_KEYWORDS`, and all other constants unrelated to Twitter quote sourcing.
- `scheduler/seed_content_data.py` — pre-analysis confirmed zero quote-account config rows; grep post-edit confirmed the same. No changes needed.
- `seed_twitter_data.py:40` has a `PeterSchiff` entry for the **twitter_agent** watchlist, which is a separate seed for a separate agent. Left alone.

**Verification:**
- `grep -rn "QUOTE_ACCOUNTS\|_search_quote_tweets" scheduler/` → zero matches
- `grep -n "twitter.com/status" scheduler/agents/content_agent.py` → zero matches
- `uv run ruff check .` → All checks passed
- `uv run pytest` → 141/141 scheduler tests pass (73 content_agent tests + the rest of the suite), including the two preserved `_draft_quote_post` tests

Net code change for this task: **+3 / −151** in `scheduler/agents/content_agent.py`. File dropped from ~1990 lines to 1845 lines.
